#+title: A TEI XML file for the Kokinwakashu combined from lexically annotated sources and bibliographically annotated sources

This repository combines two sources of TEI XML files for the Kokinwakashu, a classical Japanese poetry anthology.

The first source is a lexically annotated TEI XML file, which is a subset of the [[https://github.com/yamagen/hachidaishu][Hachidaishu Vocabulary Dateset]].

The second source is a bibliographically annotated TEI XML file, which is a digitalized version of the Karoku 2 Manuscript of the Kokinwakashu, provided by the [[https://github.com/kokubunken/kokinwakashu][嘉禄二年本『古今和歌集』の翻刻およびTEI/XMLデータ]] repository.

Note: The process of combining is assisted by Large Language Models.

* Data Files

| File | Description |
|---|---|
| =data/hachidaishu-patched.xml= | Hachidaishu lexical annotations (source) |
| =data/Kokinwakashu_200003050_20240922.xml= | Karoku 2 manuscript transcription (source) |
| =data/hachidaishu-wordlist.xml= | Extracted wordlist with dictionary entries |
| =data/kokin-merged.xml= | Karoku 2 with Hachidaishu dictionary in =<back>= |
| =data/kokin-annotated.xml= | Merged file with =<w lemmaRef>= token annotations in body |

* Tools

** wordlist — Extract lexical dictionary from Hachidaishu

Extracts all =<w>= token annotations from =data/hachidaishu-patched.xml= and
restructures them into a TEI =<back>= dictionary plus WLSPH/WLSP classification
lists. Each entry includes =<form type="inflected">= elements for all attested
conjugated forms (verbs/adjectives with 2+ distinct forms).

#+begin_src sh
cd tools/wordlist
go run ./cmd/wordlist \
  -input  ../../data/hachidaishu-patched.xml \
  -output ../../data/hachidaishu-wordlist.xml
#+end_src

Output: =data/hachidaishu-wordlist.xml= — body =<w>= elements carry =lemmaRef=
and =kanjiReading= attributes; all lexical/morphological information moves to
the back-matter =<div type="dictionary">= (1,822 entries, 314 compounds) and
two classification =<div>= elements (WLSPH and WLSP).

** merge — Embed wordlist into Karoku 2

Takes Karoku 2 as the base file and prepends the Hachidaishu dictionary and
classification lists to its existing =<back>= element. The =-sync= flag
additionally updates the =<back>= of an already-annotated file without touching
its body annotations.

#+begin_src sh
cd tools/merge
go run ./cmd/merge \
  -wordlist ../../data/hachidaishu-wordlist.xml \
  -base     ../../data/Kokinwakashu_200003050_20240922.xml \
  -output   ../../data/kokin-merged.xml \
  -sync     ../../data/kokin-annotated.xml
#+end_src

Output: =data/kokin-merged.xml= — Karoku 2 with the Hachidaishu dictionary
embedded in =<back>=, and WLSP/WLSPH taxonomy declarations added to
=<encodingDesc>=. The =-sync= target gets an updated =<back>= while preserving
all =<w lemmaRef>= body annotations.

** annotate — Batch token alignment

Walks all poems in =kokin-merged.xml=, aligns Hachidaishu tokens to Karoku 2
segment texts by exact string match, and writes =<w lemmaRef="…">= elements
into the output. Poems that cannot be fully matched are written to
=unmatched.json= for manual review.

#+begin_src sh
cd tools/annotate
go run ./cmd/annotate \
  -hachi   ../../data/hachidaishu-wordlist.xml \
  -input   ../../data/kokin-merged.xml \
  -output  ../../data/kokin-annotated.xml \
  -unmatched ../../data/unmatched.json
#+end_src

** align-review — Interactive alignment for unmatched poems

Two-subcommand tool for manually aligning poems that the batch annotator could
not match. Generates an editable draft file, opens it in Helix, then validates
and applies the alignment to the XML.

#+begin_src sh
cd tools/annotate

# Step 1: generate draft and open in Helix
go run ./cmd/align-review prepare \
  -hachi ../../data/hachidaishu-wordlist.xml \
  -input ../../data/kokin-annotated.xml \
  -poem  33

# Step 2: after editing in Helix (:wq), apply the draft
go run ./cmd/align-review apply \
  -hachi  ../../data/hachidaishu-wordlist.xml \
  -input  ../../data/kokin-annotated.xml \
  -output ../../data/kokin-annotated.xml \
  -poem   33
#+end_src

The =prepare= step reconstructs the draft from existing =<w>= annotations when
a poem has already been aligned, so re-opening a poem is safe. =lemmaRef=
values are resolved to the most specific inflected form available (e.g.
=#w.立つ.たて=).

** query — Display poem side-by-side for alignment review

Given a poem number, prints the Hachidaishu annotated tokens and the Karoku 2
surface segments in parallel, to support token alignment review.

#+begin_src sh
cd tools/query
go run . -n 1 \
  -hachi  ../../data/hachidaishu-wordlist.xml \
  -karoku ../../data/Kokinwakashu_200003050_20240922.xml
#+end_src

Poems 1–1000 have Hachidaishu annotations; poems 1001–1111 (Karoku 2 only)
show =(not found)= on the Hachidaishu side.
